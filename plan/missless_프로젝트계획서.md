# missless.co — 프로젝트 계획서

> **"그리운 사람과, 지금 바로."**
> Gemini Live Agent Challenge 2026 | Live Agent 트랙

---

## 1. 서비스 개요

### 한 줄 요약
그리운 사람의 사진·음성·대화기록을 학습한 AI가, 실시간 영상통화로 그 사람처럼 보고·듣고·말하는 서비스.

### 핵심 시나리오

| 상황 | 사용자 | 그리운 사람 |
|------|--------|------------|
| 군대 간 남자친구 | 여자친구 | 훈련소에서 연락 안 되는 남친 |
| 유학 간 딸 | 부모님 | 시차 때문에 영상통화 힘든 딸 |
| 해외 출장 중인 남편 | 아내 | 바빠서 전화 못 받는 남편 |
| 돌아가신 할머니 | 손자 | 생전 모습과 목소리를 기억하고 싶은 경우 |

### 왜 missless인가?
- **miss + less** = 그리움을 줄여준다
- 기존 GriefTech와 차별점: **죽은 사람만이 아니라, 살아있지만 지금 곁에 없는 사람** 모두가 타겟
- 감성적이면서도 밝고 긍정적인 톤

## 2. 핵심 기능 (MVP)

### Phase 1: MVP (챌린지 제출용)

#### F1. 페르소나 생성
- 사진 업로드 (3~10장) → AI가 얼굴/외형 학습
- 음성 샘플 업로드 (10~30초) → 음성 톤/억양 학습
- 카카오톡/메시지 대화 내보내기 업로드 → 말투/성격/기억 학습
- 기본 정보 입력 (이름, 관계, 성격 키워드)

#### F2. 실시간 영상통화 (Live Agent 핵심)
- Gemini Multimodal Live API 기반 양방향 실시간 대화
- 사용자: 카메라 + 마이크 ON → AI가 사용자를 보고 반응
- AI: 학습된 페르소나로 실시간 음성 응답
- 자연스러운 중단(interrupt) 지원 — 진짜 대화처럼
- 상황 인식: "거기 눈 오네?" (사용자 배경을 보고 반응)

#### F3. 상황 시뮬레이션
- "군대 간 남친과 일요일 영상통화" 같은 상황 프리셋
- 시간/장소/분위기 설정 → AI가 상황에 맞게 대화
- "크리스마스에 할머니와 통화" 등 감성 시나리오

### Phase 2: 확장 (챌린지 이후)
- 실시간 아바타 영상 생성 (얼굴이 움직이는 영상통화)
- 다국어 지원 (실시간 번역 내장)
- 그룹 통화 (여러 페르소나 동시)
- 기념일 자동 알림 ("오늘 엄마 생신이에요, 통화할까요?")

### 기술 스택 상세

| 레이어 | 기술 | 역할 |
|--------|------|------|
| **Frontend** | Next.js 14+ (App Router) | UI, WebSocket 클라이언트, 미디어 캡처 |
| **Backend** | Go + ADK for Go | 에이전트 오케스트레이션, WebSocket 서버 |
| **AI 엔진** | Gemini 2.0 Flash (Live API) | 실시간 멀티모달 대화 |
| **페르소나 학습** | Gemini Pro + System Instruction | 성격/말투/기억 커스터마이징 |
| **음성** | Gemini Live API (Native TTS) | 실시간 음성 입출력 |
| **DB** | Cloud Firestore | 페르소나 프로필, 대화 이력 |
| **파일 저장** | Cloud Storage | 사진, 음성 샘플, 채팅 로그 |
| **배포** | Cloud Run | 오토스케일링, 서버리스 |
| **인프라** | Google Cloud Platform | 전체 인프라 |

## 4. 사용자 플로우

### Flow 1: 페르소나 생성 (최초 1회)

```
[missless.co 접속]
     │
     ▼
[1. 관계 설정]
  "누구를 만나고 싶으세요?"
  → 남자친구 / 여자친구 / 부모님 / 친구 / 가족 / 기타
     │
     ▼
[2. 기본 정보]
  이름, 나이, 성격 키워드 (다정한, 장난스러운, 차분한...)
     │
     ▼
[3. 학습 자료 업로드]
  📷 사진 3~10장
  🎤 음성 메모/음성메시지 10~30초
  💬 카카오톡 대화 내보내기 (선택)
     │
     ▼
[4. AI 학습 (30초~1분)]
  "민수 님의 페르소나를 만들고 있어요..."
     │
     ▼
[5. 테스트 통화]
  짧은 30초 테스트 → "이 느낌 맞아요?" → 조정
     │
     ▼
[✅ 페르소나 완성!]
```

### Flow 2: 영상통화

```
[페르소나 선택]
  "민수에게 전화할까요?"

  🎄 크리스마스 이브 통화

  🌙 자기 전 굿나잇 콜
  ✏️ 직접 입력
     │
     ▼
[📞 통화 시작]
  ┌─────────────────────────┐
  │  ┌───────┐              │
  │  │ 상대방 │  AI 아바타    │
  │  │ 이미지 │  (사진 기반)  │
  │  └───────┘              │
  │                         │
  │  🎤 실시간 음성 대화      │
  │  👁️ AI가 내 화면을 봄    │
  │  💬 자막 표시 (선택)      │
  │                         │
  │  [통화 종료]  [상황 변경]  │
  └─────────────────────────┘
     │
     ▼

  "오늘 민수와 5분 23초 통화했어요"
  핵심 대화 하이라이트 저장
```

## 5. 심사 기준 대응 전략

### Innovation & Multimodal UX (40%) — 최중점

| 심사 포인트 | 대응 |
|------------|------|
| "텍스트 박스 탈피" | 영상통화 UI — 텍스트 입력 자체가 없음 |
| 보고, 듣고, 말하는가 | Vision(사용자 카메라) + Audio(양방향) + Speech(실시간 TTS) |
| 독특한 성격/목소리 | 페르소나 시스템 — 각 AI가 고유 성격/말투/기억 보유 |
| 실시간 & 컨텍스트 인지 | Live API 양방향 스트리밍 + 상황 시뮬레이션 |

**차별화 포인트:**
- 다른 출전작 대부분은 "AI 비서/도구"인데, missless는 **감성/관계** 중심
- "기술 데모"가 아니라 **사람이 울 수 있는 서비스** → 심사위원 감정 호소
- 데모 영상에서 실제 "군대 간 남친과의 통화" 시나리오 시연 → 강력한 스토리텔링

### Technical Implementation (30%)

| 심사 포인트 | 대응 |
|------------|------|
| GenAI SDK 또는 ADK 사용 | ADK for Go 사용 (멀티에이전트 구조) |
| Google Cloud 호스팅 | Cloud Run + Firestore + Cloud Storage |
| 할루시네이션 방지 | 페르소나 System Instruction + 기억 RAG로 그라운딩 |
| 그라운딩 증거 | 업로드된 대화/사진 기반 응답 → 출처 추적 가능 |

### Demo & Presentation (30%)

| 심사 포인트 | 대응 |
|------------|------|
| 문제와 솔루션 명확 | "보고싶은 사람, 지금 만날 수 없다면?" → missless |
| 아키텍처 다이어그램 | 위 아키텍처 다이어그램 포함 |
| 클라우드 배포 증거 | Cloud Run 콘솔 스크린샷 + 라이브 URL |

**데모 영상 구성 (3분 50초):**
